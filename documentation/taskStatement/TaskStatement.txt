# ПОСТАНОВКА ЗАДАЧИ 
Требуется написать программу состоящую из нескольких модулей. 
Между разработкой модулей существует приоритет. 
Текущий приоритет отдается модулю форматирования текста. 
Этот модуль должет проанализировать входной текст и разделить его на части по определенным правилам.
Нужно отделить друг от друга заголовки и абзацы. Они образуют отдельные части текста.
После, каждую часть нужно с определенным форматированием записать в результат.
Результатом может быть либо текстовый файл (тогда запись слабо интерактивная - подоразумевается последующее считывание данных),
либо база данных, в которой все элементы одной части отделены друг от друга (подразумевается интерактивность и возможность упрощенного считывания). 
Элементами одной части являются : номер фрагмента по порядку появления в тексте, номер отдельного предложения (пппвт), в соответствии поставленное этому номеру предложение.
В дальнейшем количество этих элементов может увеличиться. (при появлении модулей использующих эти данные)
Входной текст состоит из полностью скопированного текста из пдф файла.
Сложность его обработки в том, что каждая строка не является предложением.
Предложения размещены между строк.
Нужен алгоритм выделяющий предложения между строк.
При этом существует смысловая сложность разделения на фрагменты. 
К примеру картинок и подписей к ним, которые могут в пдф идти по тексту последовательно и тем самым разделять абзац на части, при этом создавая больше фрагментов чем нужно, нарушая тем самым некоторые предположения о структуре фрагмента \ части.
Для таких случаев нужно вынесение такого теста за пределы родного абзаца.
Например в его конец.
Так же для всего текста могут быть области, для которых не требуется разделение на части, к примеру область с ссылками на источники. 
Для этого в алгоритме разделения на части должен быть заложен механизм распознавания начала этих частей и передачи команды разделителю на части. 
Для задачи разделения на фрагменты в некоторых случаях (малый текст статьи, малое количество статей) достаточно вручную(безмодульно) добавить пустые строки для предполагаемых фрагментов текста.
В дальнейшем эту задачу может выполнить другой модуль  программы, и для больших объемов он становиться обязательным. 
После отделения фрагментов и отдельных предложений нужно записать их в отдельный файл с определенным форматированием каждого отдельного предложения.
Форматирование одинаково для всех предложений и представляет из себя размещение предложения в (стуктуре 1).
	Структура 1 :
		[st^ : ]>[end^ : ]
		Незнакомые слова (перевод) : 
		Термины (определение, ан+рус) : 
		Перевод фрагмента : 
		ИСХ > * копия исходного предложения 
		МАШ > * машинный перевод
		КОН >  *конечный перевод 
При этом для следующего модуля требуется возможность эту структуру считать (разделить внутренние части на отдельные переменные такие как ИСХ, МАШ, КОН, заполненные тестом).
Этот модуль должен будет из фрагментированного текста выделять 2 структуры ИСХ и КОН и записывать их в 2 отдельных файла при этом все предложения одного фрагменты должны быть снова объединены в один текст. 

Итог : требуется приложение функционал которого разделен на отдельные модули. Приоритет по разработке и по выпуску работоспособного кода отдается модулю по разделению текста на фрагменты. В дальнейшем возможны дополнительные улучшения : базы данных, интерфейс. В текущем исполнении требуется консольная улита с передачей 2х параметров : входного и выходного текста
